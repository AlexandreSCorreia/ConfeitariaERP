# TK-001

## Título

Cadastrar Cliente

---

## Épico

EP-001 - Cadastro de Clientes

---

## Descrição

Implementar o cadastro de clientes no sistema.

O cliente representa uma pessoa que realiza pedidos de produtos sob encomenda. O cadastro deve armazenar as informações necessárias para identificação, comunicação e entrega dos pedidos.

O cliente não possui acesso ao sistema neste momento, sendo utilizado apenas como cadastro interno para gerenciamento dos pedidos.

---

## Escopo

Implementar:

- Entidade de domínio Cliente.
- Persistência do cadastro no banco de dados.
- Endpoint para criação de clientes.
- Validação das informações recebidas.
- Tela de cadastro no frontend.
- Integração entre Angular e API.
- Registro automático da data de criação e atualização.

---

## Dados do Cadastro

### Obrigatórios

- Nome.
- Telefone.
- Endereço.

### Opcionais

- Email.
- Observações.

---

## Regras de Negócio

- O telefone deve ser único no cadastro de clientes.
- O telefone será utilizado como principal meio de comunicação com o cliente.
- O email, quando informado, deve ser único.
- O cadastro deve registrar a data de criação.
- O cadastro deve atualizar a data de alteração sempre que houver modificações.

---

## Critérios de Aceite

- Usuário consegue acessar a tela de cadastro de cliente.
- Usuário consegue informar nome, telefone, endereço, email e observações.
- Sistema impede cadastro sem nome.
- Sistema impede cadastro sem telefone.
- Sistema impede cadastro sem endereço.
- Sistema impede cadastro com telefone já existente.
- Sistema impede cadastro com email já utilizado quando informado.
- Cliente é persistido corretamente no banco de dados.
- Data de criação é registrada automaticamente.
- Data de atualização é registrada automaticamente.
- Após o cadastro, o sistema apresenta confirmação de sucesso.
- Em caso de erro, o sistema apresenta mensagem adequada.

---

## Decisões Técnicas

- O mapeamento entre Request, Entity e Response será realizado manualmente utilizando Mapper.
- As validações de backend serão implementadas utilizando FluentValidation.
- O cadastro não possui autenticação ou ativação por email neste momento.
